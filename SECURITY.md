# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to security@tuplita.ai.
Do not open a public issue for security-sensitive reports.

Useful reports include:

- The affected repository, branch, or public URL.
- Steps to reproduce the issue.
- The expected and actual behavior.
- Any known impact or exploitability details.

Tuplita will review reports through its internal vulnerability handling process.

## Scope

This repository is a static HTML and JavaScript fork of `htmlpreview`.
It does not contain backend services, authentication, secrets, deployment
credentials, or package-managed runtime dependencies.

In-scope reports include vulnerabilities in this repository's source code,
GitHub repository configuration, or GitHub Pages behavior controlled by this
repository. Vulnerabilities in third-party pages previewed through the tool are
out of scope unless they are caused by this repository.
